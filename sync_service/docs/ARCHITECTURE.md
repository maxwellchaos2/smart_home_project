# Sync Service Architecture

## 1. Роль в локальном контуре
`sync_service` работает в одном Docker Compose с `hikvision`, `logo`, `zigbee`, `scenarios`, `voice_assistant` и выступает единым шлюзом между локальным хабом и удаленным сервером.

## 2. Основные runtime-блоки
- **HTTP Proxy Layer**
	- все маршруты, кроме `/ws`, проксируются на `REMOTE_SERVER_URL`;
	- если нет заголовка `Authorization`, сервис подставляет `Bearer <authToken>`.
- **WebSocket Layer (`/ws`)**
	- локальные клиенты подключаются к `sync_service`;
	- сообщения транслируются в events репозиторий;
	- локальные изменения устройств отдельных provider-типов бродкастятся локально.
- **BLE Provisioning Layer**
	- BLE GATT сервис принимает команды от Flutter приложения;
	- поддерживает команды `wifi` и `auth_token`;
	- при успешном `auth_token` сохраняет конфиг и запускает основной режим.
- **Sync/Event Layer**
	- интеграция с `EventsModule` и `DevicesModule`;
	- отправка heartbeat устройства в удаленный контур каждые 10 секунд.

## 3. Поток первичной настройки
1. `sync_service` стартует без токена → включает BLE Gateway.
2. Flutter приложение подключается по BLE и отправляет `wifi` (`ssid`, `pass`).
3. Сервис вызывает `nmcli` и подключает малинку к Wi‑Fi.
4. Приложение отправляет `auth_token` (`value`, `device_id`, `project_id`).
5. Сервис сохраняет данные в `config_data/config.json`, останавливает BLE и запускает основной sync режим.

## 4. Поток обычной работы
1. Локальные сервисы отправляют API/WS запросы в `sync_service`.
2. Сервис проксирует их в удаленный backend с токеном устройства.
3. Для путей `/api/projects/-1/*` выполняется замена `-1` на сохраненный `projectId`.
4. События от backend и локальных источников распространяются через локальный WebSocket слой.

## 5. Границы ответственности
- Сервис не хранит пользовательские учетные данные приложения.
- Сервис хранит только технический `authToken` устройства и идентификаторы (`deviceId`, `projectId`).
- Оркестрация всех сервисов выполняется compose-слоем (`local_service/compose.yaml`), а не самим `sync_service`.
