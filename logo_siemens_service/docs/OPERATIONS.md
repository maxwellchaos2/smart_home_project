# Logo Siemens Operations

## 1. Старт
- Выполнить `dart pub get`.
- Запустить `dart run` (или make-цели, если используются).
- Подтвердить корректность сетевого доступа к Siemens LOGO.

Шаблон env:

```bash
cp .env.example .env
```

Запуск из shell:

```bash
set -a
source .env
set +a
dart run bin/server.dart
```

Ключевые переменные:
- `SYNC_SERVICE_URL`
- `MQTT_BROKER`
- `MQTT_BROKER_PORT`

## 2. Перед прод-включением
- Верифицировать карту сигналов/регистров.
- Проверить контрольные сценарии чтения/записи.
- Оценить поведение при отказах и повторных командах.

## 3. Эксплуатация
- Мониторить долю ошибок по операциям.
- Логировать изменения критичных параметров конфигурации.
- Поддерживать актуальность схем интеграции по объектам.

## 4. Диагностика
- Нет обновлений статусов: проверить ingress и подписки.
- Команды с задержкой: проверить сеть и частоту запросов.
- Ошибки маппинга: сверить версию конфигурации и mapper.

## 5. Полезные команды

Проверка доступности MQTT брокера:

```bash
nc -zv <MQTT_BROKER> <MQTT_BROKER_PORT>
```

Сборка Docker-образа:

```bash
docker buildx build --platform linux/arm64 -t logo_siemens_service:latest .
```
