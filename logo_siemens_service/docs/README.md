# Logo Siemens Service Documentation

## 1. Назначение
`logo_siemens_service` предоставляет специализированную интеграцию с Siemens LOGO в рамках текущего решения. Сервис может использоваться как отдельный слой совместимости для конкретного типа конфигураций/объектов.

Практический контекст использования:
- сервис разворачивается на локальном узле (чаще Raspberry Pi);
- по локальной сети подключается к инфраструктуре Siemens LOGO;
- применяется как продаваемая логика интеграции (продуктовый сценарий), где поведение задаётся через конфигурацию/сценарии, а не жёстко прошито в контроллере.

## 2. Ответственность
- Нормализация протокольных данных Siemens LOGO.
- Публикация состояния и прием команд управления.
- Поддержка совместимости для целевых версий конфигураций.

## 3. Отличие от `logo_controller_service`
- `logo_controller_service`: логика больше привязана к преднастроенным каналам и фиксированному поведению.
- `logo_siemens_service`: логика ориентирована на продуктовую конфигурацию, команды и реакции определяются через входящие события/сценарии и MQTT state IDs.

## 4. Обязательные переменные окружения
- `SYNC_SERVICE_URL` — URL sync/backend контура.
- `MQTT_BROKER` — адрес MQTT брокера, через который идет обмен с Siemens LOGO интеграцией.
- `MQTT_BROKER_PORT` — порт MQTT брокера (по умолчанию 1883).

## 6. Использование
Сервис включается в общий контур там, где требуется отдельный, более узкий профиль интеграции с Siemens LOGO по сравнению с универсальными контроллерными сервисами.

## 7. Что делает сервис в runtime
- Подключается к sync сервису (`SYNC_SERVICE_URL`) и получает команды/события.
- Подключается к MQTT брокеру (`MQTT_BROKER:MQTT_BROKER_PORT`).
- Подписывается на `logo/events` и получает фактические изменения состояния.
- Публикует команды в `logo/commands` для изменения состояния света.
- Конвертирует MQTT изменения в `deviceCapabilitiesChanged` события платформы.
