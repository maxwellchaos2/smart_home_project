# Local Service Documentation

## 1. Назначение
`local_service` — runtime-папка edge-контура: её копируют на сервер (обычно Raspberry Pi), затем запускают единый `docker compose`.

Здесь собрана вся локальная оркестрация сервисов и инфраструктуры, чтобы развернуть объект «под ключ» без ручной сборки каждого компонента по отдельности.

## 2. Зона ответственности
- Единый `compose.yaml` для всех локальных сервисов и их зависимостей.
- Локальная инфраструктура Zigbee/MQTT (`mosquitto`, `zigbee2mqtt`).
- Конфигурация окружения через `.env`.
- Операционные скрипты запуска и деплоя.

## 3. Что важно помнить
- Это не исходники бизнес-логики сервисов, а операционный слой их запуска.
- Папка должна переноситься на сервер целиком.
- Для стабильной работы нужны корректные host-path volume (`~/smart_home/...`) и доступ к устройствам (`/dev/ttyUSB*`, `/dev/snd`, `/dev/bus/usb`).
