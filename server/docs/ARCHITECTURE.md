# Server Architecture

## 1. Архитектурные зоны
- **API Layer**: HTTP/WebSocket интерфейсы, контрактные DTO, middleware.
- **Application Layer**: use-cases, orchestration, транзакционные операции.
- **Domain Layer**: модели, инварианты, бизнес-правила.
- **Infrastructure Layer**: БД, кэш, файловое хранилище, внешние интеграции.

## 2. Поток запроса
1. Запрос проходит аутентификацию и базовую валидацию.
2. Контроллер передает управление use-case.
3. Use-case применяет бизнес-правила и обращается к репозиториям.
4. Изменения фиксируются и при необходимости публикуются события.
5. Клиент получает нормализованный ответ.

## 3. Важные подсистемы
- Управление пользователями/ролями и правами доступа.
- Обработка устройств, комнат, сценариев и событий.
- Работа с загрузками/медиа (если используется `uploads`).
- TLS/проксирование через `nginx` и сертификаты.
