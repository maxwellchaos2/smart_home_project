# Hikvision Controller Service Documentation

## 1. Назначение
`hikvision_controller_service` обеспечивает интеграцию системы Smart Home с устройствами Hikvision Terminal. Сервис транслирует команды платформы в протоколы/эндпоинты Hikvision и нормализует получаемые статусы.

Практический контекст использования:
- сервис поднимается на локальном сервере (например, Raspberry Pi);
- работает в одной локальной сети с Hikvision terminal;
- получает команды из sync/backend контура;
- отправляет команды на терминал и принимает события от терминала.

## 2. Ответственность
- Управление соединением с внешней Hikvision-средой.
- Преобразование доменных команд платформы в команды интеграционного уровня.
- Сбор/нормализация статусов и публикация событий в общую шину.
- Защита от перегрузки/повторных запросов и обработка таймаутов.

## 4. Обязательные переменные окружения
- `SYNC_SERVICE_URL` — URL sync-сервиса (например, `http://192.168.1.10:8080`).
- `HIKVISION_IP` — IP адрес терминала Hikvision в LAN.
- `HIKVISION_USER` — логин ISAPI пользователя.
- `HIKVISION_PASSWORD` — пароль ISAPI пользователя.

Сервис читает их из `Platform.environment` или из `--define`.

## 5. Ключевые сценарии
1. Получить команду от orchestrator/backend.
2. Проверить валидность команды и состояние соединения.
3. Отправить команду во внешний контур Hikvision.
4. Принять ответ и сформировать внутренний формат события.
5. Передать результат обратно в платформу.

## 6. Что делает сервис в runtime
- Подключается к sync-сервису (`SYNC_SERVICE_URL`) через API + WS события.
- Загружает/создаёт в платформе устройство типа `doorWithLock` с `providerType = hikvision`.
- Подписывается на события `needDeviceCapabilitiesChange` и при изменении `on_off` отправляет команды открытия/закрытия двери.
- Подписывается на ISAPI event stream терминала (`/ISAPI/Event/notification/alertStream`).
- Преобразует события Hikvision в изменения capabilities/properties устройства в платформе.
- Отправляет `onDeviceHeartbeat` каждые 10 секунд.

## 7. Карта `subEventType` (Hikvision -> доменные события)
- `21` — замок разблокирован.
- `22` — замок заблокирован.
- `26` — дверь открыта.
- `27` — дверь закрыта.
- `28` — таймаут открытой двери.
- `75` — аутентификация лицом.
- `1024` — замок разблокирован из приложения.
- `1032` — input alarm.
