# LOGO Controller Operations

## 1. Запуск
- Установить зависимости: `dart pub get`.
- Поднять сервис: `dart run` или цель из `Makefile`.
- Проверить, что endpoint контроллера доступен из сети сервиса.

Шаблон env:

```bash
cp .env.example .env
```

Запуск из shell:

```bash
set -a
source .env
set +a
dart run bin/server.dart
```

## 2. Настройка
- Перед запуском задать адрес/порт/параметры протокола.
- Настроить таблицу маппинга доменных тегов.
- Проверить безопасные лимиты на частоту команд.

Ключевые параметры:
- `SYNC_SERVICE_URL`
- `LOGO_CONTROLLER_IP`
- `SNAP7_LIBRARY_PATH` (если используется нестандартный путь библиотеки)

## 3. Наблюдение
- Следить за latency операций чтения/записи.
- Контролировать процент неуспешных команд.
- Проверять консистентность входящих статусов.

## 4. Типовые проблемы
- Команды не применяются: проверить адресацию тегов/регистров.
- Данные «скачут»: проверить polling interval и дедупликацию.
- Периодические таймауты: проверить сеть, контроллер и лимиты.

## 5. Полезные команды

Проверка доступности LOGO по сети:

```bash
ping <LOGO_CONTROLLER_IP>
```

Сборка Docker-образа:

```bash
DOCKER_BUILDKIT=1 docker build --secret id=github_token,src=.github_token -t logo_controller_service:latest .
```
