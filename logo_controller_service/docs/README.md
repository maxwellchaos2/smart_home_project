# LOGO Controller Service Documentation

## 1. Назначение
`logo_controller_service` интегрирует платформу Smart Home с контроллерами Siemens LOGO!/совместимыми устройствами уровня автоматики. Сервис отвечает за отправку команд и чтение состояний с нормализацией в общую модель.

Практический контекст:
- сервис разворачивается на локальном сервере (часто Raspberry Pi);
- по локальной сети подключается к Siemens LOGO;
- используется для управления освещением (выходы LOGO) и синхронизации состояний с платформой.

## 2. Ответственность
- Коммуникация с контроллером по соответствующему промышленному протоколу.
- Маппинг входов/выходов контроллера на доменные сущности платформы.
- Гарантия корректного формата событий и команд.
- Обработка нестабильных соединений и безопасные ретраи.

## 3. Обязательные переменные окружения
- `SYNC_SERVICE_URL` — URL sync/backend контура.
- `LOGO_CONTROLLER_IP` — IP Siemens LOGO в локальной сети.
- `SNAP7_LIBRARY_PATH` — путь к `libsnap7.so` (опционально; по умолчанию системный путь контейнера/ОС).

## 5. Что делает сервис в runtime
- Подключается к sync-контуру по `SYNC_SERVICE_URL`.
- Подключается к Siemens LOGO через Snap7 (`LOGO_CONTROLLER_IP`, rack=0, slot=2).
- Слушает изменения выходов LOGO и публикует `deviceCapabilitiesChanged` в платформу.
- Слушает команды `needDeviceCapabilitiesChange` из платформы и применяет их к выходам LOGO.
- Периодически отправляет heartbeat для LOGO-устройств.

## 6. Маппинг каналов LOGO
- Выход LOGO `Q1..Q8` маппится на `providerDeviceId` вида `Q1..Q8`.
- При изменении физического выхода обновляется capability `on_off` соответствующего устройства в платформе.
- Команда `on_off` из платформы преобразуется в запись бита в нужный канал контроллера.
